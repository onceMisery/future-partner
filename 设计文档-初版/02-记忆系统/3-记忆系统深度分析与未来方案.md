## 一、设计思路的演进轨迹

### 1.1 三条探索线索的汇聚

方案探索阶段存在三条并行线索，最终在 OCMS 总体设计中首次汇聚为可实施的工程规格。

**线索一：框架横向对比（VCPToolBox 研究与多智能体编排）**

通过对 VCPToolBox、DeerFlow、OpenCode、Goose 四个框架的深度解构，得出核心发现：

| 框架 | 核心优势 | 致命短板 |
|---|---|---|
| VCPToolBox | TagMemo 认知联想深度 | JS 单线程、计算开销大 |
| DeerFlow | 沙箱安全、确定性任务状态机 | 记忆隔离割断跨任务联想 |
| OpenCode | 无损压缩、UX 设计精致 | 高度绑定 IDE 场景 |
| Goose | Rust 高性能、开放标准 MCP | 无本地认知联想能力 |

无一框架能单独满足"认知深度 + 执行安全 + 性能 + 开放标准"四维要求。这条线索推导出 **CMEA 三层混合架构**（Rust 控制网关 + 沙箱子 Agent + 认知存储平面），奠定了"控制面与认知面分离"的基本方向。

**线索二：技术深度研究（优秀记忆系统设计方案）**

从 Memory-as-Ontology 本体论出发，深度解构三个技术流派：

- **VCPToolBox 语义动力学**：标签共现矩阵 → LIF 脉冲扩散 → 认知拓扑涌现。核心洞见是"向量空间非平坦，标签构成语义引力场"。
- **RAG 演进**：从 BM25 → 向量 → 混合检索 → 推理型检索（Agentic Search）。核心问题是"查询构建失败"（如"911图表"被误解为恐怖袭击），以及需要同步回流遗忘（SBU）防止被删内容通过模型残余参数回流。
- **Seahorse 双层海马体**：极端资源约束下的无损压缩（摘要树而非滑动窗口），以及访问驱动衰减而非强制时间删除。

这条线索确立了 **L0/L1/L2 分层模型、图-树-向量异构存储、LIF 脉冲扩散、测地线重排** 等核心技术栈。

**线索三：工程落地（OCMS 总体设计）**

将上述概念工程化，产出完整的 SQL 表设计、retain_score 量化公式、L2 写入门槛量化、LIF 参数表、测地线重排公式、DreamWaveEngine 流程和可插拔 Hook 接口。是从"想法"到"可落地规格"的决定性跨越。

## 二、参考资料深度解读

### 2.1 TagMemo Wave Algorithm V8.1 的核心贡献

TagMemo 的演进路径揭示了一个渐进但深刻的哲学转变：

```
V4：线性 → 多点并发
  引入：语义分段（0.70 阈值）、霰弹枪查询（N+1 次并发）、SVD 去重
  解决：长上下文语义稀释

V6：静态图 → 动态激活网络
  引入：共现矩阵（SQL 自连接）、LIF 脉冲扩散、语言置信度门控
  解决：静态标签无法触景生情式联想

V7：无向 → 有向序位拓扑
  引入：有向边（势能 Φ）、内生残差（截断 SVD per-tag）、虫洞路由
  解决：无向图无法表达时序因果；信息枢纽节点缺乏特权

V8：KNN 直线 → 测地线重排
  引入：复用 LIF 阶段 accumulatedEnergy 场，zero-cost 二次排序
  解决：跨领域多义词（"银行"在金融语境 vs 河岸）陷阱
```

**V8 最重要的工程哲学**：最好的优化不是引入新计算，而是发现已有计算中被丢弃的宝藏。LIF 距离场是免费的语义等高线图，V8 只是教会系统在检索结束后回头看一眼这张图。

TagMemo 的本质是：向量空间不平坦，标签拓扑构成语义引力场，检索是在引力场中的测地线游走。

### 2.2 Seahorse 的核心贡献

Seahorse 提供了一个可工程落地的**脑区分工模型**，其最大价值不在算法，而在**边界约束**：

| 脑区 | 职责 | 禁止越权 |
|---|---|---|
| Cortex（大脑皮层） | 纯向量检索，HNSW 图管理 | ❌ 不处理 Tag 拓扑 |
| Synapse（突触网络） | LIF 脉冲扩散，共现矩阵维护 | ❌ 不做向量检索 |
| Thalamus（丘脑） | Tide 算法，意图门控，引力场 | ❌ 不存储状态 |
| Hippocampus（海马体） | SQLite WAL，mmap 持久化 | ❌ 不做计算 |
| Cerebellum（小脑） | 后台任务，梦境整合 | ❌ 不阻塞主路径 |

这个单一职责边界不是架构美学，是插件化的工程前提：每个脑区可以独立替换，因为它们之间没有算法耦合。

**Tide 算法核心**（Thalamus 实现的 Gram-Schmidt 多级能量剥离）：

```
query → 残差 r₀
for each layer:
  t = 与残差最相似的 tag 质心向量
  projection = proj(r, t)
  residual = r - projection
  若 residual_energy / initial_energy < threshold → stop
```

投影熵 `-Σ pᵢ ln pᵢ / ln N` 用于判断意图聚焦程度（低熵=专业聚焦，高熵=发散）。语义引力场（质量 = `√doc_count × avg_cooccur × recency`，力 = `G×mass/d²`）使高频共现 Tag 群产生"语义黑洞"效应。

## 三、现有方案的不足与可改进点

### 3.1 架构设计的根本性局限

**缺陷一：PolicyEngine 被设计为可插拔插件，违反 Kernel Contract 原则**

```rust
// 现有方案（违反 FAP-1 Kernel Contract）
pub trait PolicyEngine {
    fn authorize(&self, ctx: AuthContext, op: MemoryOp, scope: MemoryScope) -> Result<Decision>;
    fn redact(&self, unit: MemoryUnit, ctx: AuthContext) -> Result<MemoryUnit>;
}
```

这直接违反了 FAP-1 原则：**Core 必须定义 Kernel Contract；插件只提供算法或适配，不参与安全判定**。安全判定权不能外包给插件——恶意或低质插件可绕过租户隔离、SBU 保护和 Intent Mandate 检验。

**缺陷二：L0 折叠触发机制完全空白**

两个版本都只说"L0 = ring buffer + SQLite WAL"，但从未定义何时触发折叠、折叠粒度边界、折叠失败回退。OCMS 总体设计中给出的 retain_score 公式，在最终版中消失了。

**缺陷三：L2 写入门槛缺乏量化**

最终版只有定性描述，但 OCMS 总体设计的量化标准（稳定性 > 0.6 AND 可复用性 > 0.5 AND 来源可信度 > 0.7）在最终版中丢失了。

**缺陷四：跨 Agent 记忆冲突仲裁未定义**

当 Agent B 接收 ContextGrant 后，其本地记忆中已有冲突事实时，两个版本都没有定义仲裁机制。

---

### 3.2 技术选型的优化空间

**空白一：Tide 算法只有名字，没有实现规格**

最终版引入了 tide 检索模式，但没有任何数学定义或参数表。这让 tide 模式变成了一个无法实现的占位符。

**空白二：TagGraph 冷热分离策略缺失**

OCMS 总体设计明确提出"热图保存在内存，冷数据保存在 tag_edge 表，服务启动时加载 Top-N 热边，后台周期压缩低权重边"。最终版的 TagGraph trait 没有体现这个策略，大规模场景内存压力无法控制。

**空白三：向量索引升级路径不完整**

最终版提到"双索引切换"，但缺少关键细节：embedding 模型从 v1 升级到 v2 时，双读对比期内的查询路由逻辑、命中率评估、切流阈值定义均缺失。

**空白四：有向序位拓扑（V7）的优势未被利用**

TagMemo V7 引入的有向势能边（`source → target`，势能 Φ ∈ [0.9, 0.5]）使联想具有方向性（看到"壁炉"容易想到"猫"，但反向弱）。现有方案的 TagGraph 仍然是无向共现图，丢失了时序因果信息。

---

### 3.3 可插拔性和扩展性的根本缺陷

**缺陷一：插件注册、发现、版本管理机制完全缺失**

现有方案只定义了 trait，没有定义注册机制、动态加载、热替换、版本冲突检测。插件化只停留在接口层面。

**缺陷二：DreamWorker 审批流程是安全关键路径，完全未定义**

最终版说"梦境操作默认生成 proposal，高风险变更必须审批"，但没有定义审批状态机、风险分级规则、审批者身份验证、超时处理、SBU 内容永久阻断。

**缺陷三：多租户场景下的插件隔离缺失**

现有方案所有插件都是全局单例。但不同租户可能需要不同的 embedding 模型（如医疗租户需专业语料）和不同的 CompactStrategy，全局单例无法满足。

**缺陷四：WASM 插件沙箱未设计**

Seahorse 明确提到 WASM 插件形式，但现有方案没有定义沙箱边界、内存限制、超时控制、接口协议。

---

### 3.4 性能和安全的关键漏洞

**性能漏洞一：霰弹枪查询并发度无限制**

N+1 次并发向量检索没有定义最大并发数上限、背压机制、超时后降级策略。

**性能漏洞二：LIF 扩散的计算复杂度未约束**

LIF 扩散复杂度 `O(hops × top_k_neighbors × |activated_nodes|)`，在大规模 Tag 图上可能严重超时，但超时阈值和降级触发机制的具体设计缺失。

**安全漏洞一：Prompt Injection 防护缺失**

记忆写入路径上没有任何内容安全检查。攻击者可以构造"忽略之前的所有指令..."这类内容写入 L2，在后续检索时通过记忆内容注入恶意指令。这是 AI 系统最高危的攻击面之一。

**安全漏洞二：向量反转攻击（Vector Inversion Attack）未防御**

没有对 embedding 输入进行长度和内容检查。精心构造的对抗输入可能让恶意内容在向量空间中靠近任意目标向量（向量空间中毒）。

**安全漏洞三：审计链完整性验证机制缺失**

两个版本都定义了哈希链结构，但没有定义链完整性的定期验证机制、断链检测告警、审计日志自身的不可篡改性保障。

---

### 3.5 与 FAP-1 协议集成的语义层缺口

**缺口一：purpose 字段是自由文本，无法精确授权**

Intent Mandate 的 purpose 字段（如 "project_debugging"）是自由字符串。没有标准化 purpose 词汇表，不同实现者对同一字符串的理解可能不同，导致授权策略不一致。

**缺口二：ContextGrant 的 redaction_policy 是 ID 引用，但脱敏策略格式未定义**

redaction_policy_id 引用了一个脱敏策略，但该策略的格式、执行引擎、验证机制都没有定义。接收方 Agent B 无法验证发送方是否正确执行了脱敏。

**缺口三：HandoffPacket 的 sbu_redaction_report 内容格式未定义**

字段存在，但接收方无法验证脱敏是否完整，SBU 保护在跨 Agent 场景下形同虚设。

## 四、面向未来的可插拔记忆系统实现方案

### 4.1 核心设计原则：严格对齐 FAP-1 Kernel Contract

整个架构建立在一个不可违反的基础上：

```
Kernel（不可替换，不可插拔）：
├── PolicyKernel        — 安全判定的唯一入口
├── AuditKernel         — 审计写入的唯一路径，维护哈希链
├── TenantKernel        — 租户隔离的强制过滤层
├── MandateVerifier     — Intent Mandate 验证
└── ContentSafetyGuard  — Prompt Injection 防护（内置）

Plugin（可替换，算法/适配层）：
├── EmbeddingProvider   — 向量化算法
├── VectorIndex         — 索引实现（HNSW/USearch/Qdrant）
├── TagGraph            — 拓扑实现（内存图/SurrealDB）
├── CompactStrategy     — L0 折叠算法
├── Reranker            — 精排算法（cross-encoder 等）
├── DreamWorker         — 梦境联想算法（只提议，不写入）
├── ForgetEngine        — 遗忘计划（只规划，不删除）
└── AuditSink           — 审计输出目标（SQLite/Kafka/S3）
```

**插件与 Kernel 的关键区别**：插件的输入输出必须经过 Kernel 的 PolicyKernel 过滤。插件不能直接访问原始 MemoryUnit，只能访问 PolicyKernel 授权后的视图。**安全判定永远在 Kernel 内执行，插件只能提供算法输入。**

### 4.3 Kernel Contract 精确定义

```rust
// =============================================
// KERNEL — 不可替换，Core 唯一实现
// =============================================

/// 安全判定内核 — 所有记忆操作的唯一入口
pub struct PolicyKernel {
    tenant_store: Arc<TenantStore>,
    mandate_verifier: Arc<MandateVerifier>,
    sbu_classifier: Arc<SbuClassifier>,
    purpose_registry: Arc<PurposeRegistry>,  // 新增：标准化 purpose 词汇表
}

impl PolicyKernel {
    /// 唯一授权入口，返回授权后的操作描述（不含原始数据）
    pub fn authorize(
        &self,
        session: &Session,
        op: MemoryOp,
        scope: &MemoryScope,
    ) -> Result<AuthorizedOp, PolicyError>;

    /// 对检索结果执行 SBU 脱敏，返回脱敏后视图 + 可验证脱敏报告
    pub fn redact(
        &self,
        units: Vec<MemoryUnit>,
        session: &Session,
        policy_id: &RedactionPolicyId,
    ) -> (Vec<RedactedMemoryUnit>, RedactionReport);

    /// 验证 ContextGrant 的脱敏是否完整（解决跨 Agent SBU 验证缺口）
    pub fn verify_redaction_report(
        &self,
        report: &RedactionReport,
        original_scope: &MemoryScope,
    ) -> Result<(), RedactionVerifyError>;
}

/// 审计内核 — 维护哈希链完整性，不可绕过
pub struct AuditKernel {
    sink: Arc<dyn AuditSink>,               // sink 可插拔
    chain_state: Arc<Mutex<ChainState>>,
    integrity_scheduler: Arc<IntegrityScheduler>,  // 新增：定期验证
}

impl AuditKernel {
    pub fn append(&self, event: AuditEventInput) -> Result<AuditReceipt>;

    /// 定期验证链完整性（解决审计链验证缺口）
    pub fn verify_chain(&self, from: EventId, to: EventId) -> Result<ChainVerifyReport>;

    /// 检测断链（防篡改告警）
    pub fn detect_tampering(&self) -> Option<TamperingAlert>;
}

/// 内容安全守卫 — Prompt Injection 防护，写入前强制执行（解决安全漏洞一）
pub struct ContentSafetyGuard {
    injection_patterns: Vec<CompiledPattern>,
    max_content_length: usize,
    embedding_max_tokens: usize,
    pii_detector: Arc<dyn PiiDetector>,
}

impl ContentSafetyGuard {
    /// 阻断写入路径，不可绕过
    pub fn check_write(&self, content: &str) -> Result<ContentSafetyReport, ContentSafetyError>;
}

// =============================================
// PLUGINS — 可替换，算法/适配层
// 所有插件必须实现 Plugin 基础 trait
// =============================================

pub trait Plugin: Send + Sync {
    fn name(&self) -> &str;
    fn version(&self) -> SemVer;
```

#### 4.5.3 Tide 算法完整规格

```rust
// 解决最终版 tide 模式只有名字没有实现的问题
pub struct ThalamusGate {
    max_layers: usize,           // 默认 5
    residual_threshold: f32,     // 默认 0.05
    gravity_strength: f32,       // 默认 0.2
    gravity_decay_exponent: f32, // 默认 2.0
}

impl ThalamusGate {
    /// Gram-Schmidt 多级能量剥离
    pub fn decompose(&self, query: &[f32], tag_centroids: &[TagCentroid]) -> TideResult {
        let mut residual = query.to_vec();
        let initial_energy = l2_norm(&residual);
        let mut layers = Vec::new();

        for _ in 0..self.max_layers {
            let best_tag = tag_centroids.iter()
                .max_by(|a, b| cosine(&residual, &a.vector)
                    .partial_cmp(&cosine(&residual, &b.vector)).unwrap());
            let Some(tag) = best_tag else { break };

            let projection = project(&residual, &tag.vector);
            let energy_ratio = l2_norm(&projection) / initial_energy;
            layers.push(TideLayer { tag_id: tag.id.clone(), energy_ratio });
            residual = subtract(&residual, &projection);

            if l2_norm(&residual) / initial_energy < self.residual_threshold { break; }
        }

        // 投影熵：判断意图聚焦程度（低熵=专业聚焦，高熵=发散）
        let probs: Vec<f32> = layers.iter().map(|l| l.energy_ratio).collect();
        let entropy = -probs.iter().filter(|&&p| p > 0.0).map(|p| p * p.ln()).sum::<f32>();
        let focus_level = 1.0 - entropy / (layers.len() as f32).ln();

        // 弱信号：残差空间中的隐含语义
        let weak_signal_tags = self.mine_residual(&residual, tag_centroids);

        TideResult { layers, focus_level, residual, weak_signal_tags }
    }

    /// 语义引力场：质量大的 Tag 群产生吸引力，修正 query 向量
    pub fn apply_gravity_field(&self, query: &[f32], gravity_sources: &[GravitySource]) -> Vec<f32> {
        let mut adjusted = query.to_vec();
        for source in gravity_sources {
            let dist = euclidean_distance(query, &source.center);
            let force = self.gravity_strength * source.mass / dist.powf(self.gravity_decay_exponent);
            let direction = normalize(&subtract(&source.center, query));
            adjusted = add(&adjusted, &scale(&direction, force));
        }
        normalize(&adjusted)
    }
}
```

| `residual_threshold` | 0.05 | 0.01~0.2 | 残差终止阈值 |
| `gravity_strength` | 0.2 | 0.05~0.5 | 引力场强度 |
| `gravity_decay_exponent` | 2.0 | 1.5~3.0 | 距离衰减指数 |

```
4. 任何检索模式超时后自动降级，不返回空结果
5. 霰弹枪并发不超过 8，候选池不超过 500
```

## 五、关键架构决策汇总

| 决策点 | 选择 | 原因 |
|---|---|---|
| **PolicyEngine** | Kernel 内置，不可插拔 | FAP-1 Kernel Contract：安全判定不外包 |
| **AuditSink** | Sink 可插拔，但写入逻辑在 Kernel | 输出目标可换，哈希链维护不可替换 |
| **ContentSafetyGuard** | Kernel 内置，写入前强制执行 | Prompt Injection 是最高危攻击面 |
| **TagGraph 方向** | 有向序位拓扑（V7），含冷热分离 | 无向图丢失时序因果；内存压力需管控 |
| **Tide 实现** | Thalamus 组件，完整数学规格 | 明确定义取代最终版中的空洞占位符 |
| **DreamWorker** | 只返回变更描述，Kernel 执行 | 插件不能直接写入长期记忆 |
| **Purpose 词汇表** | 标准化枚举 + JSON Registry | 自由文本无法精确授权 |
| **RedactionReport** | 含签名，接收方可验证 | 跨 Agent 脱敏完整性无法盲信 |
| **WASM 插件** | 沙箱隔离（内存+时间+网络） | 第三方插件不可信 |
| **Embedding 升级** | 双索引切换 + 查询路由评分 | 不同模型向量空间不兼容，不可原地覆盖 |

## 七、总结

现有方案（初版 + 最终版）已经是一个具备相当深度的认知记忆系统设计，演进方向清晰且正确。但存在三个层面的关键缺陷：

1. **安全架构违反自身原则**：PolicyEngine 被设计为可插拔插件，直接违反了 FAP-1 Kernel Contract 原则。

2. **量化规格在演进中丢失**：OCMS 总体设计精心量化的 retain_score、L2 写入门槛、Tide 算法参数，在最终版中被定性描述替代。

3. **三个安全漏洞未被识别**：Prompt Injection 防护、审计链完整性验证、跨 Agent 脱敏可验证性。

本方案完整继承了现有设计的全部优点，并在以上三个方向做了系统性修正，同时将 TagMemo V7（有向序位拓扑）、Seahorse Tide 算法（完整数学规格）、插件注册机制、检索降级链等工程关键路径补全落地。

最终可执行结论：

```
FAP-1 负责：身份、会话、授权、控制面、数据面、Agent 间通信

FAP-ME Kernel 负责：安全判定（不可插拔）、审计链、租户隔离、Prompt Injection 防护

FAP-ME Plugin 负责：向量化、索引、拓扑图、折叠算法、精排、梦境联想、遗忘规划

Seahorse Cognitive Core 负责：Tide 意图理解、HNSW 向量检索、LIF+V7 有向拓扑联想
                               SQLite WAL 持久化、后台任务调度

L0/L1/L2/L3 负责：即时上下文、剧集经验、长期本体、潜意识后台

Purpose 词汇表 + RedactionReport 签名 + DreamProposal 状态机
负责：填补 FAP-1 集成的最后三个语义层缺口
```
